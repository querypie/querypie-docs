Overview

다른 외부 응용프로그램에서 사용자를 인증하고 데이터에 접근할 수 있도록 QueryPie가 OAuth 2.0 Provider 역할을 수행할 수 있습니다. 외부 응용프로그램을 클라이언트로 등록하고 QueryPie가 인증 토큰을 발급하여 안전하게 쿼리파이의 리소스에 접근할 수 있도록 합니다.

OAuth Client Application 등록

Admin > General > System > Integrations 의 Authentication 하위에 있는 OAuth Client Application

  • Admin > General > System > Integrations 의 Authentication 하위에 있는 OAuth Client Application 항목을 선택합니다.

  • +Add버튼을 누르고 등록할 애플리케이션의 정보를 입력한 뒤 Save 버튼을 누릅니다.

    • Name : 식별이 용이한 이름을 입력합니다.

    • Redirect URI : 인증 과정이 끝난 후 돌려보낼 Client URL을 입력합니다.

    • Access Token Timeout (Minutes) : 기본값은 60, 최소값은 5입니다.

    • Refresh Token Timeout (Minutes) : 기본값은 1440, 최소값은 5입니다.

Add OAuth Client Application

  • 설정이 저장된 후 출력되는 팝업에 Client ID와 Client Secret이 출력됩니다. Client Secret은 최초 생성된 값을 다시 확인할 수 없으므로 복사해서 잘 보관해야 합니다.

OAuth Client Application 설정 수정

OAuth Client Application 설정 수정

OAuth Client Application 목록에서 행을 클릭하면 다음의 작업을 할 수 있습니다.

  • 설정 값 변경 : Name, Redirection URI, Access Token 및 Refresh Token Timeout

  • Regenerate Client Secret : 최초 생성했던 Client Secret을 버리고 새로 생성할 수 있습니다.

OAuth Client Application 설정 삭제

등록한 OAuth Client Application 설정을 삭제하려면 목록에서 삭제 대상 항목을 체크하고 Delete 버튼을 누릅니다.
