## Overview

Administrators can grant or revoke roles (Roles) with server access permissions to users or user groups.

## Granting Roles

### 1. Select the target to grant permissions to.

<figure data-layout="center" data-align="center">
![Administrator &gt; Servers &gt; Server Access Control &gt; Access Control](/administrator-manual/web-apps/web-app-access-control/access-control/granting-and-revoking-roles/image-20250629-161238.png)
<figcaption>
Administrator &gt; Servers &gt; Server Access Control &gt; Access Control
</figcaption>
</figure>


1. Navigate to Administrator &gt; Web Apps &gt; Web App Access Control &gt; Access Control menu.
2. Select the user or user group to grant permissions to.

### 2. Select the Role to grant permissions to.

<figure data-layout="center" data-align="center">
![image-20250629-161326.png](/administrator-manual/web-apps/web-app-access-control/access-control/granting-and-revoking-roles/image-20250629-161326.png)
</figure>

1. Select the Roles tab on the left side of the screen.
2. Click the `+ Grant Roles` button in the center right.
3. Check the checkbox on the left of the Role to grant.
4. Enter the Expiration Date. The default value is 1 year.
5. Click the `Grant` button.


## Revoking Roles

<figure data-layout="center" data-align="center">
![image-20250629-161437.png](/administrator-manual/web-apps/web-app-access-control/access-control/granting-and-revoking-roles/image-20250629-161437.png)
</figure>

1. Navigate to Administrator &gt; Web Apps &gt; Web App Access Control &gt; Access Control menu.
2. Select the user or user group to grant permissions to.
3. Go to the Roles tab.
4. Select the Role to revoke permissions from in the Role list. (Multiple selection possible)
5. Click the `Revoke` button displayed in the top left of the list.
6. Enter Revoke in the confirmation popup and click the `Revoke` button to successfully revoke the Role.


<Callout type="info">
**Q. Is there a place where I can check the history of Role grants or revocations?**
A. You can check it by selecting the Web Apps &gt; Web App Role History menu in the Audit menu.
</Callout>
