## Overview

에이전트 없이 URL Proxy를 통한 접속 또한 지원합니다. Redash, Tableau, Zepplin 과 같은 웹 서비스에서 DB에 연결하고자 할 때, QueryPie에서 생성된 URL Proxy 인증 정보를 통해 접근 통제 정책을 적용받으면서도 안전하게 데이터베이스에 접근이 가능합니다.


## URL Proxy 인증 정보 수령하기

<Callout type="info">
이 작업은 관리자에 의해 Proxy 사용이 활성화된 DB 커넥션에 한해서만 수행이 가능합니다. 사용 설정을 위해 [Database Proxy 사용 활성화](Database Proxy 사용 활성화) 문서를 참고하여 주시기 바랍니다.
* 10.2.6부터 Oracle 사용자 커넥션 페이지 Proxy Info에 Service Name/SID 정보가 표기됩니다.
</Callout>

<figure data-layout="center" data-align="center">
![QueryPie Web &gt; Data Access Control &gt; Databases &gt; Proxy Info](/user-manual/database-access-control/connecting-to-proxy-without-agent/image-20240731-133408.png)
<figcaption>
QueryPie Web &gt; Data Access Control &gt; Databases &gt; Proxy Info
</figcaption>
</figure>

1. QueryPie 로그인 후 상단 메뉴 바 옵션에서 `Databases`를 클릭합니다.
2. 우측 상단의 `Proxy Info`를 클릭합니다. 
3. 사용하고자 하는 서비스에  **Proxy Host** 및  **Proxy Port** 를 복사하여 실제 DB에 대한 호스트 대신 기입합니다. 
4. 하단의 `Create Token` 버튼을 클릭하여 사용자 인증 토큰을 발급받습니다. 

<figure data-layout="center" data-align="center">
![QueryPie Web &gt; Data Access Control &gt; Databases &gt; Proxy Info &gt; Create Token](/user-manual/database-access-control/connecting-to-proxy-without-agent/image-20240731-133805.png)
<figcaption>
QueryPie Web &gt; Data Access Control &gt; Databases &gt; Proxy Info &gt; Create Token
</figcaption>
</figure>

1. 해당 팝업창에서 제공하는  **Username** 및  **Password** 를 사용하고자 하는 서비스에 기입하여 쿼리파이 프록시를 통한 연결을 활성화 합니다. (해당 사용자 인증 토큰 정보는 최초 발급 시에만 전체 노출됩니다)
2. 만약 토큰 키를 분실한 경우 기존 키를 내역에서 `Revoke` 버튼을 클릭하여 삭제하고, 다시 `Create Token`버튼을 클릭하여 토큰을 새로 발급 받아 사용하시면 됩니다.
