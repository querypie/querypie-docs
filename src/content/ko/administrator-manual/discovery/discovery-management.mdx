## Overview

민감정보의 위치를 특정하고 민감정보를 식별하기 위해서는 먼저 Discovery job을 생성하고 수행해야 합니다. 이는 관리자페이지의 Discovery 페이지에서 시작할 수 있습니다.

## Discovery 관리 페이지로 이동

1. 사용자 대시보드에서 관리자 페이지로 이동합니다.<br/> 
2. 상단 메뉴에서 “Discovery”를 선택합니다.

## Discovery 메뉴 구성

Discovery Management 는 아래와 같은 매뉴로 구성되어 있습니다.

<figure data-layout="center" data-align="center">
![image-20240728-203328.png](/administrator-manual/discovery/discovery-management/image-20240728-203328.png)
</figure>


* Dashboard : 발견된 민감 정보에 대해 관리자가 확인하여 확정된 결과를 반영한 통계정보를 다양한 위젯 통해 시각적으로 표현합니다. 
* Inventory : 발견된 민감 정보에 대해 관리자가 확인하여 확정된 결과를 반영하여 각각의 Data source 별, Data path 별로 정보를 확인한 수 있습니다.
* Discovery Jobs : 민감정보를 찾고 분류하기 위해서는 먼저 Discovery job을 생성해야 합니다. 이 Job 을 생성, 수정, 삭제와 관련된 관리를 수행합니다. 특정 Job을 수동으로 시작하거나 수행중인 Job을 강제 중지 할 수 있습니다.
* Discovery History : Job이 수행된 이력과 발견된 민감정보의 이력 그리고 관리자가 확인한 이력을 확인(예정)할 수 있습니다.
* Scan Results : 관리자는 발견된 민감정보에 대해 정 / 오를 구별하여 오탐일 경우 오탐임을 표시하여 inventory에 최종 반영할 결과를 선택할 수 있습니다.
* Detection Profiles : Data pattern 들의 집합입니다. 기본적으로 민감정보의 유형별, 관련된 컴플라이언스별 built-in profile이 제공됩니다.
* Data Patterns : 디스커버리에 사용할 데이터 패턴 목록입니다. 사용자 편의를 위해 사전정의된 패턴을 제공하고 있으며 사용자가 원하는 정규식을 입력하여 사용자 정의 할 수도 있습니다.
