## Overview

Password Provisioning 을 통해 패스워드가 변경된 서버의 계정을 조회하고, 패스워드 변경 이력을 확인하고, 개별 서버 계정의 패스워드를 수동으로 관리할 수 있습니다.

<Callout type="info">
이 메뉴는 Administrator &gt; General &gt; Company Management &gt; [Security](https://chequer.atlassian.net/wiki/spaces/QM/pages/544178422/Security#%EC%84%9C%EB%B2%84-%EC%BB%A4%EB%84%A5%EC%85%98-%EB%B3%B4%EC%95%88-%EC%84%A4%EC%A0%95)에서 Server Connection Security의  **Password Provisioning**  설정을 On으로 선택 후 저장해야 표시됩니다.
</Callout>

## Account 조회하기

<figure data-layout="center" data-align="center">
![Administrator &gt; Servers &gt; Server Account Management &gt; Account Management ](/administrator-manual/servers/server-account-management/account-management/Screenshot-2025-07-27-at-6.47.45-PM.png)
<figcaption>
Administrator &gt; Servers &gt; Server Account Management &gt; Account Management 
</figcaption>
</figure>

1. Administrator &gt; Servers &gt; Server Account Management &gt; Account Management 메뉴로 이동합니다
2. 테이블 좌측 상단의 검색란을 통해 서버 또는 Account의 검색이 가능합니다.
3. 테이블 우측 상단의 새로고침 버튼을 통해 계정 목록을 최신화할 수 있습니다.
4. 테이블에서 이하의 컬럼 정보를 제공합니다.
    1.  **Server** : 대상 서버 명
    2.  **Host**  : 서버의 IP
    3.  **Server OS**  : 서버의 OS
    4.  **Account**  : 서버 접속용 Account
    5.  **Usage**  : 계정의 사용 용도
        1.  **For User** : 사용자 접속 용도 계정
        2.  **For Provisioning**  : QueryPie의 패스워드 변경 Job에서 사용되는 계정
    6.  **Last Password Updated At :**  계정의 마지막 패스워드 업데이트 일시
    7.  **Created At**  : 계정 최초 변경 일시
5. 각 행을 클릭하면 계정 세부 정보 조회가 가능합니다.

## One Time Account 조회하기

<figure data-layout="center" data-align="center">
![Administrator &gt; Servers &gt; Server Account Management &gt; Account Management &gt; One Time Accounts 탭](/administrator-manual/servers/server-account-management/account-management/Screenshot-2025-07-27-at-6.50.29-PM.png)
<figcaption>
Administrator &gt; Servers &gt; Server Account Management &gt; Account Management &gt; One Time Accounts 탭
</figcaption>
</figure>

1. Administrator &gt; Servers &gt; Server Account Management &gt; Account Management &gt; One Time Accounts 탭으로 이동합니다
2. 테이블 좌측 상단의 검색란을 통해 Server, Host 또는 Account의 검색이 가능합니다.
3. 테이블 우측 상단의 새로고침 버튼을 통해 계정 목록을 최신화할 수 있습니다.
4. 테이블에서 이하의 컬럼 정보를 제공합니다.
    1.  **Server** : 대상 서버 명
    2.  **Host**  : 서버의 IP
    3.  **Server OS**  : 서버의 OS
    4.  **Account**  : 서버 접속을 위해 생성된 One Time Account
    5.  **Last Fail Message**  : 삭제된 원인을 담은 에러 에러메세지
    6.  **Last Delete Attempted At**  : 계정 삭제를 마지막으로 시도한 일시
    7.  **Delete Attempt Counts**  : 계정 삭제 시도 횟수

## Active Directory 조회하기

<figcaption>
Administrator &gt; Servers &gt; Server Account Management &gt; Account Management &gt; Active Directory탭
</figcaption>
</figure>

1. Administrator &gt; Servers &gt; Server Account Management &gt; Account Management &gt; Active Directory 탭으로 이동합니다
2. 테이블 좌측 상단의 검색란을 통해 UPN의 검색이 가능합니다.
3. 테이블 우측 상단의 새로고침 버튼을 통해 계정 목록을 최신화할 수 있습니다.
4. 테이블에서 이하의 컬럼 정보를 제공합니다.
    1.  **AD Name**  : QueryPie에 연동된 AD 이름
    2.  **AD Domain**  : AD 도메인 주소
    3.  **UPN**  : 해당 도메인에 로그인할 때 사용하는 사용자 계정
    4.  **Last Password Updated At**  : 계정의 마지막 패스워드 업데이트 일시
    5.  **Created At**  : 계정 최초 변경 일시

## Account의 세부 정보 조회하기

<figure data-layout="center" data-align="center">
![image-20240831-155640.png](/administrator-manual/servers/server-account-management/account-management/image-20240831-155640.png)
</figure>

1. Administrator &gt; Servers &gt; Server Account Management &gt; Account Management 메뉴로 이동합니다
2. 확인하려는 account가 있는 행을 클릭합니다.
3. Log Details을 통해 패스워드 변경 이력을 확인합니다.
4. `View Password` 버튼을 클릭하여 현재 패스워드를 확인할 수 있습니다.

<Callout type="info">
View Password를 하기 위해선 본인인증 과정(패스워드 재입력)을 수행합니다. QueryPie의 User 접속시 OTP가 적용되어 있을 경우, View Password에서도 OTP 인증을 수행합니다.
</Callout>
