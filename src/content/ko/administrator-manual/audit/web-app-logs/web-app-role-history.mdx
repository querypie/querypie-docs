## Overview

조직에서 관리하는 웹 애플리케이션 역할 부여 및 회수 이력을 기록합니다.

## Web App Role History 조회하기

<p align="center">
![Screenshot-2025-06-30-at-2.47.18-PM.png](/administrator-manual/audit/web-app-logs/web-app-role-history/Screenshot-2025-06-30-at-2.47.18-PM.png)
*Administrator &gt; Audit &gt; Web Apps &gt; Web App Role History*
</p>

1. Administrator &gt; Audit &gt; Web Apps &gt; Web App Role History 메뉴로 이동합니다.
2. 현재 월 기준으로 로그가 내림차순으로 조회됩니다.
3. 테이블 최측 상단의 검색란을 통해 아래와 조건으로 검색이 가능합니다.<br/>a. User Name : 사용자 이름<br/>b. User Email : 사용자 이메일<br/>c. Role Name : 역할명
4. 검색 필드 우측 필터 버튼을 클릭하여 AND/OR 조건으로 아래와 필터링이 가능합니다.
    1.  **Event**  : 이벤트 유형
    2.  **User Type**  : 사용자 유형
    3.  **Action At ** : 액션 발생 일시 범위
5. 테이블 우측 상단의 새로고침 버튼을 통해 로그 목록을 최신화할 수 있습니다.
6. 테이블에서 아래와 같은 정보를 제공합니다:
    1.  **No**  : 순번
    2.  **Action At**  : 액션 발생 일시
    3.  **Event**  : 이벤트 유형 (Role Granted, Role Revoked)
    4.  **User Type**  : 사용자 유형 (USER, GROUP)
    5.  **Name**  : 사용자명
    6.  **Email**  : 사용자 이메일
    7.  **Role Name**  : 부여/회수된 역할명
    8.  **Expiration Date**  : 역할 만료 일시
    9.  **Action By**  : 액션을 수행한 관리자명

## Web App Role History 상세 내역 조회하기

각 행을 클릭하면 세부 정보 조회가 가능합니다.
<p align="center">
![Screenshot-2025-06-30-at-2.50.17-PM.png](/administrator-manual/audit/web-app-logs/web-app-role-history/Screenshot-2025-06-30-at-2.50.17-PM.png)
*Administrator &gt; Audit &gt; Web Apps &gt; Web App Role History &gt; Web App Role History Details*
</p>

*  **우측 드로워에는 아래와 정보를 노출합니다:** 
    1.  **Role Name**  : 부여/회수된 역할명
    2.  **Event**  : 이벤트 유형 (Role Granted, Role Revoked)
    3.  **Name**  : 대상 사용자 이름
    4.  **Email**  : 대상 사용자 이메일
    5.  **Action At**  : 액션 시점
    6.  **Action By**  : 액션을 수행한 관리자명
    7.  **Expiration Date**  : 역할 만료 일시
    8.  **Policies**  : 해당 역할에 포함된 정책 목록
        1. Name : 정책명
        2. Description : 정책 설명
        3. Version : 정책 버전
