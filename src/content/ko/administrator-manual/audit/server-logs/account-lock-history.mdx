## Overview
QueryPie의 사용자별 서버 접속 권한 잠김 이력을 표시합니다. 패스워드 오입력, 비인가 IP의 접속 시도 등 접속 실패한 횟수를 모두 카운트 하며, 관리자가 설정한 접속 실패 임계치 도달시 접속 계정이 Lock 상태가 됩니다.

## Server Account Lock History 조회하기
<p align="center">
<div>[image-20240728-175941.png]()</div>
*Administrator &gt; Audit &gt; Servers &gt; Server Account Lock History*
</p>

1. Administrator &gt; Audit &gt; Servers &gt; Server Account Lock History 메뉴로 이동합니다.
2. 테이블에 내림차순으로 목록에 로그가 노출됩니다.
3. 테이블 좌측 상단의 검색란을 통해 이하의 조건으로 검색이 가능합니다.
    1. **Name**  : 사용자 이름
    2. **Email**  : 사용자 Email
    3. **Account**  : 서버 접속 Account
    4. **Role Name**  : 사용된 Role 명
    5. **Server Name**  : 접속 서버 명
4. 검색 필드 우측 필터 버튼을 클릭하여  **Action At** 에 대한 일시 범위를 재지정하여 필터링이 가능합니다.
    1. **Event**  : 이벤트 유형
        1. **Account Locked**  : 사용자의 계정 잠김 이벤트
        2. **Account Unlocked**  : 관리자의 계정 잠김 해제 이벤트
    2. **Action At**  : 이벤트 발생 일시 범위
5. 테이블 우측 상단의 새로고침 버튼을 통해 로그 목록을 최신화할 수 있습니다.
6. 테이블에서 이하의 컬럼 정보를 제공합니다:
    1. **No**  : 이벤트 식별 번호
    2. **Action At**  : 이벤트 발생 일시
    3. **Event**  : 이벤트 유형
        1. **Account Locked**  : 사용자의 계정 잠김 이벤트
        2. **Account Unlocked**  : 관리자의 계정 잠김 해제 이벤트
    4. **Name**  : 대상 사용자/그룹 이름
    5. **Email**  : 대상 사용자 이메일
        1. 그룹의 경우에는 “-”으로 표기합니다.
    6. **Role Name**  : 부여/회수된 Role 이름
    7. **Server Group**  : 서버 그룹 명
    8. **Server Name**  : 서버 명
    9. **Account**  : Lock 대상 서버의 Account
    10. **Action By**  : 이벤트를 발생한 사용자 또는 관리자명
