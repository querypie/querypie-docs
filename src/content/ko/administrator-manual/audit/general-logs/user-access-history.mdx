## Overview
모든 사용자의 QueryPie 시스템 접근과 관련된 모든 이력, 즉 계정 로그인/아웃, 계정의 상태 변경, 비밀번호 재설정 메시지 전송 등의 이력을 조회할 수 있습니다.

## User Access History 조회하기
<p align="center">
<div>[image-20240714-063443.png]()</div>
*Administrator &gt; Audit &gt; General &gt; User Access History*
</p>

1. Administrator &gt; Audit &gt; General &gt; User Access History 메뉴로 접근합니다.
2. 이번 달의 로그가 최신 순으로 조회됩니다.
3. **Action Type**  : 시스템 접근 관련 행위 유형을 나타냅니다. 일부 로그는 Activity Logs에도 동시에 기록됩니다.
    * **User Login**  : 사용자의 QueryPie 로그인
    * **User Logout ** : 사용자의 QueryPie 로그아웃
    * **Account Locked**  : 비밀번호 연속 오류로 인한 계정 잠김
    * **Account Expired ** : 사용자 활동 없음으로 인한 계정 만료 (Activity Logs에 ** User Deactivated**  로 기록)
    * **Account Locked Manually**  : 관리자에 의해 특정 사용자 계정을 잠금 (Activity Logs에  **User Updated**  로 기록)
    * **Account Unlocked**  : 관리자에 의해 특정 사용자 계정을 다시 활성화함 (Activity Logs에  **User Reactivated ** 로 기록)
    * **Password Reset Push ** : 관리자에 의해 사용자 계정의 비밀번호 재설정이 발생됨
4. **Result**  : 행위의 결과를 표시합니다.
5. **Message**  : 필요시, 행위의 구체적인 내용을 기록합니다. Message 컬럼의 텍스트로 로그를 검색할 수 있습니다.
