## Overview

쿼리파이 사용자/그룹에 부여되는 DB 커넥션에 대한 접근 권한 부여 및 회수 이력을 기록합니다.

## DB Access Control Logs 조회하기

<figure data-layout="center" data-align="center">
![Administrator &gt; Audit &gt; Databases &gt; Access Control Logs](/administrator-manual/audit/database-logs/access-control-logs/image-20240730-072419.png)
<figcaption>
Administrator &gt; Audit &gt; Databases &gt; Access Control Logs
</figcaption>
</figure>

1. Administrator &gt; Audit &gt; Databases &gt; Access Control Logs 메뉴로 이동합니다.
2. 당월 기준으로 내림차순으로 로그가 조회됩니다.
3. 테이블 좌측 상단의 검색란을 통해 이하의 조건으로 검색이 가능합니다.
    1.  **Name**  : 사용자 이름
    2.  **Connection Name**  : DB 커넥션명
    3.  **Email**  : 사용자 Email
    4.  **Privilege Name**  : 사용자 접속 권한명
    5.  **DB Host**  : DB 호스트
4. 검색 필드 우측 필터 버튼을 클릭하여 AND/OR 조건으로 이하의 필터링이 가능합니다.
    1.  **Connection Name**  : DB 커넥션명
    2.  **Database Type ** : 데이터베이스 유형
    3.  **Event**  : 권한 부여/회수 이벤트 유형
        1.  **Connection Owner Granted**  : Connection 오너 권한 부여 이력
        2.  **Connection Owner Revoked**  : Connection 오너 권한 회수 이력
        3.  **Access Control Granted**  : 커넥션 접근 권한 부여 이력
        4.  **Access Control Updated**  : 커넥션 접근 권한 수정이력
        5.  **Access Control Revoked**  : 커넥션 접근 권한 회수 이력
    4.  **Action At**  : 권한 부여/회수 일시 범위
5. 테이블 우측 상단의 새로고침 버튼을 통해 로그 목록을 최신화할 수 있습니다.
6. 테이블에서 이하의 컬럼 정보를 제공합니다:
    1.  **No**  : 이벤트 식별 번호
    2.  **Action At**  : 권한 부여/회수 일시
    3.  **Event**  : 권한 부여/회수 이벤트 유형
    4.  **User Type**  : 사용자/그룹 유형
    5.  **Name**  : 대상 사용자/그룹 이름
    6.  **Email**  : 대상 사용자 이메일
        1. 그룹의 경우에는 미표기합니다.
    7.  **Privilege**   **Name**  : 사용자 접속 권한
    8.  **Description**  : 재활성화 등 입력 사유 출력
    9.  **Connection Name**  : 대상 DB 커넥션명
    10.  **Database Type**  : 대상 데이터베이스 유형
    11.  **Replication Type**  : DB Replication 유형
    12.  **DB Host**  : 접속한 DB 호스트
    13.  **Action By**  : Permission 부여/회수를 진행한 관리자명 또는 System
