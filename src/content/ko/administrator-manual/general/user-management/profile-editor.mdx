## Overview

관리자는 사용자 아이덴티티의 효율적인 원장 관리를 위한 각 프로필 속성(Attribute) 별 Attribute-Level Mastering의 관리 구현이 가능합니다. Profile Editor를 통해 사용자의 각 속성 별로 원장 주체를 설정하고 이에 대한 업데이트 관리가 가능합니다. 해당 Profile Editor의 경우, QueryPie 사용자에 대해서는 영향도가 없으며 Auth Provider가 Okta, LDAP 등 외부 IdP로 지정되어 있는 사용자의 속성 관리 주체를 통제합니다.

또한, 미리 정의된 속성뿐만 아니라 조직의 필요에 따라 사용자 정의 속성(Custom Attribute)을 생성, 수정, 삭제하는 것도 가능합니다. 이를 통해 각 조직의 정책에 맞춘 유연한 사용자 정보 관리가 가능합니다.

## 사용자 속성 주체 설정 정의

각 사용자 속성 별 Source Priority를 지정할 수 있으며, 종류는 아래와 같습니다:

1.  **Inherit from profile source** 
    1. 외부 Auth Provider로부터 해당 Attribute에 대한 업데이트를 수신을 허용하겠다는 의미의 옵션입니다. 
    2. SCIM API를 통해 사용자의 Attribute가 업데이트(patch) 가능한 상태가 됩니다. 
2.  **Inherit from QueryPie** 
    1. 외부 Auth Provider로부터 생성된 사용자이지만, 해당 Attribute는 쿼리파이 자체적으로 관리하겠다는 의미의 옵션입니다. 
    2. SCIM API를 통해 사용자의 해당 Attribute가 업데이트 푸시되어도 쿼리파이 자체적으로 관리하는 Attribute가 되어 변경을 무시합니다. 

## Status 컬럼 정의

Profile Editor의 Status 컬럼은 해당 Attribute가 사용자 프로필 (General &gt; Users &gt; User Detail &gt; Profile)에 표시될지 여부를 설정합니다.

1. 조작 방식: 드롭다운 메뉴를 통해 `Visible` 또는 `Hidden`을 선택합니다.
    * `Visible`: 사용자 프로필에 해당 Attribute가 표시됩니다.
    * `Hidden`: 사용자 프로필 및 사용자 프로필 수정 화면(User Profile Edit)에서도 해당 Attribute가 표시되지 않습니다.
2. 적용 대상: 기본 Attribute와 Custom Attribute 모두에 적용 가능합니다.
3. 상시 Visible: 다음 기본 Attribute는 항상 `Visible` 상태로 유지되며 `Hidden`으로 변경할 수 없습니다. 변경 시도 시 " 'Variable Name' is managed by the system and cannot be changed."와 같은 오류 메시지가 표시됩니다.
    * Username
    * Primary email
    * Display name
    * Risk score
4. 제한 사항 : IdP와 연동된 Attribute의 Status는 수정할 수 없습니다.
