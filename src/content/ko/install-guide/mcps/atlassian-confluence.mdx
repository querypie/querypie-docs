# Atlassian Confluence

Atlassian Confluence와 연결하여 팀 협업 및 문서화 플랫폼을 관리할 수 있습니다.

## 주요 기능
- 페이지 및 블로그 포스트 관리
- 스페이스 및 권한 관리
- 콘텐츠 검색 및 조회
- 댓글 및 협업 기능
- 첨부파일 관리

## 기본 정보

### 인증 방식
API Token을 사용하여 인증합니다.

### 입력 파라미터
- **API Token** (필수): Confluence에서 생성한 API 토큰
- **Base URL** (필수): Confluence 인스턴스 URL
- **Username** (필수): Confluence 사용자명

## 지원하는 MCP 도구

### get_spaces
모든 스페이스 목록을 반환합니다. 결과는 ID 오름차순으로 정렬되며, `limit` 파라미터로 결과 수를 제한할 수 있습니다.

### get_pages
모든 페이지 목록을 반환합니다. 결과 수는 `limit` 파라미터로 제한할 수 있으며, 추가 결과는 `Link` 응답 헤더의 `next` URL을 통해 확인할 수 있습니다.

### get_page_by_id
특정 페이지의 상세 정보를 조회합니다.

### bulk_user_lookup
요청 본문에 제공된 ID들에 대한 사용자 세부 정보를 반환합니다.

### create_page
스페이스에 새로운 페이지를 생성합니다.

### update_page
ID로 페이지를 업데이트합니다.

### delete_page
ID로 페이지를 삭제합니다.

## 사용 예시

예를 들어,
- Prompt로 "특정 스페이스의 페이지 목록을 조회해줘" 라고 입력하면,
- LLM은 MCP를 통해 `get_pages` 도구를 호출하여 페이지 목록을 조회합니다.
- 그리고, Tool 호출 결과를 기반으로 채팅에서 응답합니다.

## API Token 생성 방법
1. Confluence 설정 → Personal Access Tokens 이동
2. "Create token" 클릭하여 새 토큰 생성
3. 필요한 권한을 설정합니다.

## 주의사항
- 생성한 API Token은 안전하게 보관하세요.
- 필요한 최소한의 권한만 부여하는 것을 권장합니다.
- Base URL은 HTTPS를 사용하는 것을 권장합니다.
