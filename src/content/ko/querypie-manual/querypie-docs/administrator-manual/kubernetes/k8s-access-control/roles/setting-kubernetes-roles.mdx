## Overview
쿠버네티스에 대한 사용자 접근을 허용하거나 제한하기 위해 조직 내에서의 쿠버네티스 역할을 기반으로 하는 역할 기반 액세스 제어(Role-Based Access Control, RBAC)를 적용합니다. 사용자나 사용자 그룹에게 설정한 Role을 부여할 수 있습니다. Role은 여러 정책을 종합하여 단일 역할로 정의하는 데 사용됩니다.

## Role 정책 할당하기 
Administrator > Kubernetes > K8s Access Control > Roles > List Details

1. Administrator > Kubernetes > K8s Access Control > Roles 메뉴로 이동합니다.
2. Roles 목록에서 대상 Role 행을 클릭하여 상세 페이지로 이동합니다.
3. Policies 탭 우측의 + Assign Policies 버튼을 클릭합니다.
4. 할당할 정책을 검토한 뒤, 좌측의 체크박스를 체크합니다. Policy명으로 검색 가능합니다.기존에 이미 할당되어 있는 정책은 체크박스가 비활성화됩니다.리스트 목록에는 각 정책별 이하의 정보를 노출합니다:Name : Policy명 정책 정보를 조회할 수 있는 모달 링크를 제공합니다.
  1. Policy명으로 검색 가능합니다.
  2. 기존에 이미 할당되어 있는 정책은 체크박스가 비활성화됩니다.
  3. 리스트 목록에는 각 정책별 이하의 정보를 노출합니다:Name : Policy명 정책 정보를 조회할 수 있는 모달 링크를 제공합니다.
    * Name : Policy명 정책 정보를 조회할 수 있는 모달 링크를 제공합니다.
      * 정책 정보를 조회할 수 있는 모달 링크를 제공합니다.
5. Assign 버튼을 누르면 체크박스를 체크한 대상 정책을 할당합니다.
6. (Cancel 버튼을 누르면 해당 모달을 변경사항 없이 종료합니다.)

## Role 정책 할당 해제하기  
1. Administrator > Kubernetes > K8s Access Control > Roles 메뉴로 이동합니다.
2. Roles 목록에서 대상 Role 행을 클릭하여 상세 페이지로 이동합니다.
3. Policies 탭에서 전체 선택 또는 개별 선택 박스에 체크 시 컬럼 바에 노출된 Unassign 버튼을 클릭합니다.
4. 컨펌창에서 Unassign 버튼 클릭 시, 선택한 아이템이 할당 해제되어 리스트에서 사라집니다.
5. (Cancel 버튼 클릭 시 컨펌창만 닫습니다.)

Policy 할당 해제 시 해당 정책으로 사용자 및 그룹에 부여되어 있던 모든 접근 권한이 회수됩니다.
