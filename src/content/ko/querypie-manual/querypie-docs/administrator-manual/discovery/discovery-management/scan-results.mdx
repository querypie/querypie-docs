## Overview
관리자는 발견된 민감정보에 대해 정 / 오를 구별하여 오탐일 경우 오탐임을 표시하여 inventory에 최종 반영할 결과를 선택할 수 있습니다.

Administrator > Discovery > Discovery Management > Scan Results

관리자는 Scan Results 페이지에서 발견된 민감정보의 오탐 여부를 판단하고 accept 또는 false positive 중 하나를 선택합니다. accept하면 결과(발견된 대상에 sensitive item tag를 부여)를 inventory에 반영합니다. false positive를 선택하면 대상을 false positive로 기록합니다.  

## 발견된 민감정보의 경로 확인 및 검토
Scan Result 목록

민감정보가 위치한 경로와 민감정보의 유형 관련된 컴플라이언스를 목록에서 확인할 수 있습니다. 

* Export : 목록의 내용을 JSON 형식으로 내보낼 수 있습니다. 쿼리파이 설정에서 암호화가 강제되어 있으면 zip 압축파일에 암호를 입력해야 합니다.
* Verify : 발견된 민감정보에 대한 내용을 확인하고 최종 판단을 저장할 수 있습니다.

## 스캔 결과에 대한 검토
1. 민감정보가 있는 것으로 판단되는 data path 마다 verify 버튼을 클릭하여 상세 페이지로 이동 합니다.
2. 상세페이지에서  처리되기 전의 항목은 New Items 목록에 나열됩니다.Verify item Confidence : 발견된 항목의 95%신뢰구간 수치를 계산하여 Low / Medium / High 의 3단계로 표시합니다. 정규식의 경우(이항분포)의 예를 들자면, 발견된 항목이 100개 중 32개 이하면 Low, 33~66개면 Medium, 67개 이상일 경우 High입니다. 이 수치로 AI identifier 또는 정규식의 검출 수준을 판단할 수 있습니다. Verification : Accept / False Positive 둘 중 하나를 선택할 수 있습니다. 결과가 정탐이라면 Accept, 오탐이라면 False Positive를 선택합니다. Accept를 하면 태그가 확정되어 Inventory에 반영됩니다. False Positive를 선택하면 상태가 False Positive로 기록되고 이는 Inventory의 상세 화면의 “Sensitive Items에서도 확인할 수 있습니다. New Items 목록의 행을 클릭하면 아래 그림과 같은 상세 화면을 볼 수 있습니다. 대상에 대한 상세정보 해당 민감정보를 발견하는데 사용한 Detection profile이 무엇인지 추출에 사용된 패턴이 무엇인지 확인할 수 있습니다.
  1.  Confidence : 발견된 항목의 95%신뢰구간 수치를 계산하여 Low / Medium / High 의 3단계로 표시합니다. 정규식의 경우(이항분포)의 예를 들자면, 발견된 항목이 100개 중 32개 이하면 Low, 33~66개면 Medium, 67개 이상일 경우 High입니다. 이 수치로 AI identifier 또는 정규식의 검출 수준을 판단할 수 있습니다. 
  2. Verification : Accept / False Positive 둘 중 하나를 선택할 수 있습니다. 결과가 정탐이라면 Accept, 오탐이라면 False Positive를 선택합니다. Accept를 하면 태그가 확정되어 Inventory에 반영됩니다. False Positive를 선택하면 상태가 False Positive로 기록되고 이는 Inventory의 상세 화면의 “Sensitive Items에서도 확인할 수 있습니다.
  3.  New Items 목록의 행을 클릭하면 아래 그림과 같은 상세 화면을 볼 수 있습니다. 대상에 대한 상세정보 해당 민감정보를 발견하는데 사용한 Detection profile이 무엇인지 추출에 사용된 패턴이 무엇인지 확인할 수 있습니다.
3. Accept 또는 False Positive를 선택하면 아래의 Verification Result로 항목이 이동됩니다.Verification Reult
4. Description에 해당 결정에 대한 코멘트를 기록할 수 있습니다.
5. Save 버튼을 눌러 저장을 하면 최종적으로 Inventory에 반영됩니다.
