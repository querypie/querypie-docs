## Overview
Data pattern 들의 집합입니다. 기본적으로 민감정보의 유형별, 관련된 컴플라이언스별 built-in profile이 제공됩니다.사용자는 Data pattern을 선택하여 프로파일을 사용자 정의할 수 있습니다.

## Built-in(사전 정의된) Detection Profile
<p align="center">
<div>[image-20240730-203304.png]()</div>
</p>

Built-in (사전 정의된) Detection profile은 크게 민감정보 유형별, 컴플라이언스별 두가지가 있습니다.

* Types of Sensitive ItemsPII : Personal Identifiable InformationPHI : Protected Health InformationNPI : Non-Public InformationCHD : Cardholder DataSAD : Sensitive Authentication Data
  * PII : Personal Identifiable Information
  * PHI : Protected Health Information
  * NPI : Non-Public Information
  * CHD : Cardholder Data
  * SAD : Sensitive Authentication Data
* ComplianceGDPR : General Data Protection Regulation (EU)HIPAA : Health Insurance Portability and Accountability Act (US)ISMS-P : Personal information & Information Security Management System (South Korea)PCI-DSS : Payment Card Industry Data Security StandardPIPA : Personal Information Protection Act (South Korea, 개인정보보호법)HITECH : Health Information Technology for Economic and Clinical Health Act (US)PIPEDA : Personal Information Protection and Electronic Documents Act (Canada)GLBA : Gramm-Leach-Bliley Act (US)CCPA : California Consumer Privacy Act (US California)
  * GDPR : General Data Protection Regulation (EU)
  * HIPAA : Health Insurance Portability and Accountability Act (US)
  * ISMS-P : Personal information & Information Security Management System (South Korea)
  * PCI-DSS : Payment Card Industry Data Security Standard
  * PIPA : Personal Information Protection Act (South Korea, 개인정보보호법)
  * HITECH : Health Information Technology for Economic and Clinical Health Act (US)
  * PIPEDA : Personal Information Protection and Electronic Documents Act (Canada)
  * GLBA : Gramm-Leach-Bliley Act (US)
  * CCPA : California Consumer Privacy Act (US California)

<Callout type="important">
사전 정의된 Detection Profile을 선택하여 discovery job을 생성할 수 있지만 연결되어 있는 data pattern들이 많기 때문에 스캔 시간이 오래 걸릴 수 있습니다. 빠른 스캔을 수행해야 하는 상황이라면 사용자 정의 프로파일을 생성하여 원하는 패턴만 선택하여 스캔을 수행하는 것을 권장 드립니다. 
</Callout>

<Callout type="info">
Built-in profile은 삭제하거나 수정할 수 없지만 어떤 패턴을 포함하고 있는지 내용을 볼 수 있습니다. 
</Callout>

## 사용자 정의 프로파일 추가
Detection profile 목록 상단의 + Detection Profile을 버튼을 누르면 생성 페이지로 이동되고 사용자가 검출에 사용되는 data patten을 지정하여 Detection Profile을 생성할 수 있습니다.

<p align="center">
<div>[image-20240730-204829.png]()</div>
</p>

* Data pattern 추가+ Data Pattern 버튼을 누르면 지정할 패턴 목록이 팝업되고 하나이상의 패턴을 선택하여 추가할 수 있습니다.데이터에 대한 탐색을 수행하기 위해서는 (Data) 유형의 패턴을 지정합니다.경로이름만 탬색을 수행하기 위해서는 (Keyword) 유형의 패턴을 지정합니다.AI Identifier를 사용할 수 있는 항목 Address - Korea : 대한민국 주소Country code - ISO 3166-1 alpha-2 and ISO 3166-1 alpha-3 : 알파벳 2 ~ 3글자로 이루어진 국가코드Disease name - Korean : 한글로 작성된 질병명Job Title - Korean : 한글로 작성된 직업명License / Certificate - Korean : 한글로 작성된 자격증 이름Name - Korean : 한글 이름Romanized name - Korean : 로마자 표기된 한글이름
  * + Data Pattern 버튼을 누르면 지정할 패턴 목록이 팝업되고 하나이상의 패턴을 선택하여 추가할 수 있습니다.
  * 데이터에 대한 탐색을 수행하기 위해서는 (Data) 유형의 패턴을 지정합니다.
  * 경로이름만 탬색을 수행하기 위해서는 (Keyword) 유형의 패턴을 지정합니다.
  * AI Identifier를 사용할 수 있는 항목 Address - Korea : 대한민국 주소Country code - ISO 3166-1 alpha-2 and ISO 3166-1 alpha-3 : 알파벳 2 ~ 3글자로 이루어진 국가코드Disease name - Korean : 한글로 작성된 질병명Job Title - Korean : 한글로 작성된 직업명License / Certificate - Korean : 한글로 작성된 자격증 이름Name - Korean : 한글 이름Romanized name - Korean : 로마자 표기된 한글이름
    * Address - Korea : 대한민국 주소
    * Country code - ISO 3166-1 alpha-2 and ISO 3166-1 alpha-3 : 알파벳 2 ~ 3글자로 이루어진 국가코드
    * Disease name - Korean : 한글로 작성된 질병명
    * Job Title - Korean : 한글로 작성된 직업명
    * License / Certificate - Korean : 한글로 작성된 자격증 이름
    * Name - Korean : 한글 이름
    * Romanized name - Korean : 로마자 표기된 한글이름

* Data pattern 제거추가가 완료된 목록에서 특정 패턴을 지정하여 제거할 수 있습니다.
  * 추가가 완료된 목록에서 특정 패턴을 지정하여 제거할 수 있습니다.
* Detection Profile Name : Detection Profile 이름을 입력합니다.
* Description : 사용자 정의한 Detection Profile에 대한 설명을 입력합니다. Job에 연결할 때 어떤 패턴들의 집합인지 식별하기 용이하게 하려면 설명을 상세히 작성하는 것이 도움이 될 수 있습니다.

## 사용자 정의 프로파일 편집 및 삭제
* 사용자 정의 프로파일 편집 : Detection Profile 목록에서 특정 행을 클릭하면 설정된 내용을 확인하고 수정할 수 있습니다.
* 사용자 정의 프로파일 삭제 : 특정 사용자 정의 프로파일을 체크하고 Delete 버튼을 클릭합니다.

<Callout type="important">
사용자 정의 detection profile이 어떤 discovery job에 사용되고 있다면 detection profile을 삭제할 수 없습니다. 삭제하기 위해서는 먼저 discovery job에서 해당 detection profile을 제외해야 합니다. 
</Callout>
