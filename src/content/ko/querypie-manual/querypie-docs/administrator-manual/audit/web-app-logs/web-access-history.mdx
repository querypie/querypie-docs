## Overview
조직에서 관리하는 웹 애플리케이션 역할 이력을 기록합니다.

## Web Access History 조회하기
<p align="center">
<div>[image-20250629-171743.png]()</div>
*Administrator &gt; Audit &gt; Web Apps &gt; Web Access History*
</p>

1. Administrator &gt; Audit &gt; Web Apps &gt; Web Access History 메뉴로 이동합니다.
2. 현재 월 기준으로 로그가 내림차순으로 조회됩니다.
3. 테이블 최측 상단의 검색란을 통해 아래와 조건으로 검색이 가능합니다.
    1. **User Name**  : 사용자 이름
    2. **User Email**  : 사용자 Email
    3. **Role Name**  : 역할명
    4. **Web App Name**  : 웹 애플리케이션 이름
    5. **Web App Base Url**  : 웹 애플리케이션 기본 URL
    6. **Session Uuid**  : 세션 고유 식별자
    7. **Tab Id ** : 탭 식별자
    8. **Client Ip**  : 클라이언트 IP 주소
    9. **Message**  : 메시지 내용
4. 검색 필드 우측 필터 버튼을 클릭하여 AND 조건으로 아래와 필터링이 가능합니다.
    1. **Action Type**  : 액션 유형
    2. **Action At**  : 액션 발생 일시 범위

## Web Access History 상세 내역 조회하기
각 행을 클릭하면 세부 정보 조회가 가능합니다.

<p align="center">
<div>[image-20250629-172248.png]()</div>
*Administrator &gt; Audit &gt; Web Apps &gt; Web Access History &gt; Web Access History Details*
</p>

* **우측 드로워에는 아래와 정보를 노출합니다:**
    1. **Action At**  : 액션 시점
    2. **Action Type**  : 액션 유형 (Connect / Disconnect)
    3. **Result**  : 접속 성공/실패 여부
        1. ✅ Success
        2. ❌ Failure
    4. **Role**  : 사용자 역할명
    5. **Message**  : 액션 실행 등 특이사항에 대한 기록
    6. **Name**  : 대상 사용자 이름
    7. **Email**  : 대상 사용자 이메일
    8. **Client IP**  : 클라이언트 IP 주소
    9. **Web App**  : 웹 애플리케이션명
    10. **Base URL**  : 웹 애플리케이션 기본 URL
    11. **Session ID**  : 세션 고유 식별자
    12. **Tab ID**  : 탭 식별자
