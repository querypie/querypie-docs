## Overview
QueryPie의 사용자별 커넥션 패스워드 실패 및 장기미접속으로 인한 커넥션 잠금 이력을 표시합니다. 패스워드 오입력, 비인가 IP의 접속 시도 등 접속 실패한 횟수를 모두 카운트 하며, 관리자가 설정한 접속 실패 임계치 도달시 접속 계정이 Lock 상태가 됩니다.

## DB Account Lock History 조회하기
Administrator > Audit > Databases > Account Lock History

1. Administrator > Audit > Databases > Account Lock History 메뉴로 이동합니다.
2. 테이블에 발생 시점 내림차순으로 목록에 로그가 노출됩니다.
3. 테이블 좌측 상단의 검색란을 통해 사용자 이름으로 검색이 가능합니다.
4. 검색 필드 우측 필터 버튼을 클릭하여 Action At에 대한 일시 범위를 재지정하여 필터링이 가능합니다.Connection Name : 접속한 DB 커넥션명Database Type : 데이터베이스 유형Lock Status : 잠금 유형Locked : 사용자의 계정 잠금 이벤트Unlocked : 관리자의 계정 잠금 해제 이벤트Action At : 이벤트 발생 일시 범위
  1. Connection Name : 접속한 DB 커넥션명
  2. Database Type : 데이터베이스 유형
  3. Lock Status : 잠금 유형Locked : 사용자의 계정 잠금 이벤트Unlocked : 관리자의 계정 잠금 해제 이벤트
    1. Locked : 사용자의 계정 잠금 이벤트
    2. Unlocked : 관리자의 계정 잠금 해제 이벤트
  4. Action At : 이벤트 발생 일시 범위
5. 테이블 우측 상단의 새로고침 버튼을 통해 로그 목록을 최신화할 수 있습니다.
6. 테이블에서 이하의 컬럼 정보를 제공합니다:Name : 대상 사용자 이름Email : 대상 사용자 이메일Connection Name : 대상 DB 커넥션명Database Type : 대상 데이터베이스 유형Attempted Count : 접속 시도 횟수Lock Status : 잠금 현황Unlocked By : 잠금해제 수행자 정보Unlocked At : 접속 계정 잠금 해제 일시Locked At : 접속 계정 잠금 일시Action At : 이벤트 발생 일시Host Name : 접속 실행 호스트명Client IP : 사용자 클라이언트 IP 주소Replication Type : DB Replication 유형DB Host : 접속한 DB 호스트DB Name : DB명
  1. Name : 대상 사용자 이름
  2. Email : 대상 사용자 이메일
  3. Connection Name : 대상 DB 커넥션명
  4. Database Type : 대상 데이터베이스 유형
  5. Attempted Count : 접속 시도 횟수
  6. Lock Status : 잠금 현황
  7. Unlocked By : 잠금해제 수행자 정보
  8. Unlocked At : 접속 계정 잠금 해제 일시
  9. Locked At : 접속 계정 잠금 일시
  10. Action At : 이벤트 발생 일시
  11. Host Name : 접속 실행 호스트명
  12. Client IP : 사용자 클라이언트 IP 주소
  13. Replication Type : DB Replication 유형
  14. DB Host : 접속한 DB 호스트
  15. DB Name : DB명
