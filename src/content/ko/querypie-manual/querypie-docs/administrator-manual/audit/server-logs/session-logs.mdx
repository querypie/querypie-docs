## Overview
조직에서 관리하는 서버 접속 세션을 레코딩합니다. 관리자는 영상 재생을 통하여 서버내에서 사용자의 작업 수행 이력을 모니터링할 수 있습니다.

## Session Logs 조회하기
Administrator > Audit > Servers > Session Logs

1. Administrator > Audit > Servers > Session Logs 메뉴로 이동합니다.
2. 테이블에 내림차순으로 목록에 로그가 노출됩니다.
3. 테이블 좌측 상단의 검색란을 통해 이하의 조건으로 검색이 가능합니다.Name : 사용자 이름Email : 사용자 EmailAccount : 서버 접속 AccountServer Name : 접속 서버 명Host : 접속 서버 호스트Client IP : 사용자 IPClient Name : 사용자의 접속 방식
  1. Name : 사용자 이름
  2. Email : 사용자 Email
  3. Account : 서버 접속 Account
  4. Server Name : 접속 서버 명
  5. Host : 접속 서버 호스트
  6. Client IP : 사용자 IP
  7. Client Name : 사용자의 접속 방식
4. 검색 필드 우측 필터 버튼을 클릭하여 Action At에 대한 일시 범위를 재지정하여 필터링이 가능합니다.Server OS : 접속한 서버의 OSProtocol : 접시 사용한 ProtocolAction At : 접속 시점Connected From : 접속 방식
  1. Server OS : 접속한 서버의 OS
  2. Protocol : 접시 사용한 Protocol
  3. Action At : 접속 시점
  4. Connected From : 접속 방식
5. 테이블 우측 상단의 새로고침 버튼을 통해 로그 목록을 최신화할 수 있습니다.
6. 테이블에서 이하의 컬럼 정보를 제공합니다:No : 이벤트 식별 번호Action At : 세션 녹화 시작 일시Name : 대상 사용자 이름Email : 대상 사용자 이메일Role : 서버 접속시 사용된 Role명Account : 서버 접속 AccountServer Name : 접속 서버 명Server OS : 접속한 서버의 OSHost : 접속 서버 호스트Port : 접속시 사용된 포트Protocol : 접시 사용한 ProtocolClient IP : 사용자 클라이언트 IP 주소Client Name : 사용자의 접속 방식Connected From : 연결 방식의 주체WebProxy
  1. No : 이벤트 식별 번호
  2. Action At : 세션 녹화 시작 일시
  3. Name : 대상 사용자 이름
  4. Email : 대상 사용자 이메일
  5. Role : 서버 접속시 사용된 Role명
  6. Account : 서버 접속 Account
  7. Server Name : 접속 서버 명
  8. Server OS : 접속한 서버의 OS
  9. Host : 접속 서버 호스트
  10. Port : 접속시 사용된 포트
  11. Protocol : 접시 사용한 Protocol
  12. Client IP : 사용자 클라이언트 IP 주소
  13. Client Name : 사용자의 접속 방식
  14. Connected From : 연결 방식의 주체WebProxy
    1. Web
    2. Proxy

## Session Record 재생하기
각 행의 Action At의 링크를 클릭하면 세션 레코딩 내역의 재생이 가능합니다.

녹화된 파일의 크기가 700MB를 초과여부에 따라 재생화면을 노출하거나 다운로드 버튼을 노출합니다.

1. 700MB 미만상단에 기본 정보가 노출되며 하단에 Replay 재생화면이 노출됩니다.Name: 실행자명Action At: 발생 일시 (시작)Session Record: 레코딩명재생버튼을 누르면 시작됩니다.반대로, 일시정지 버튼을 누르면 재생이 멈춥니다.버튼 우측 라인을 드래그하거나 원하는 시점을 클릭하여 해당 부분부터 재생도 가능합니다.Close 버튼을 누르면 창이 닫힙니다.
  1. 상단에 기본 정보가 노출되며 하단에 Replay 재생화면이 노출됩니다.Name: 실행자명Action At: 발생 일시 (시작)Session Record: 레코딩명
    1. Name: 실행자명
    2. Action At: 발생 일시 (시작)
    3. Session Record: 레코딩명
  2. 재생버튼을 누르면 시작됩니다.반대로, 일시정지 버튼을 누르면 재생이 멈춥니다.버튼 우측 라인을 드래그하거나 원하는 시점을 클릭하여 해당 부분부터 재생도 가능합니다.
    1. 반대로, 일시정지 버튼을 누르면 재생이 멈춥니다.
    2. 버튼 우측 라인을 드래그하거나 원하는 시점을 클릭하여 해당 부분부터 재생도 가능합니다.
  3. Close 버튼을 누르면 창이 닫힙니다.
2. 700MB 이상재생 화면 안에 실행 불가 문구와 함께 다운로드 버튼을 제공합니다.파일 크기가 700MB를 초과하여 세션을 재생할 수 없습니다.이 세션 녹화를 보려면 파일 콘텐츠를 다운로드하여 플레이해야 합니다.Download 버튼을 누르면 다운로드가 진행되어 로컬에 파일이 생성됩니다.Administrator > General > Company Management > Security > Others > Export a file with Encryption 공통 옵션이 Required로 활성화 되어 있을 경우, 해당 다운로드 시 암호화 모달이 추가로 나타나고 해당 모달 이후에 다운로드를 실행합니다.
  1. 재생 화면 안에 실행 불가 문구와 함께 다운로드 버튼을 제공합니다.파일 크기가 700MB를 초과하여 세션을 재생할 수 없습니다.이 세션 녹화를 보려면 파일 콘텐츠를 다운로드하여 플레이해야 합니다.
