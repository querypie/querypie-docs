# Emailを通じたユーザーパスワードリセット

### Overview

QueryPie 10.2.2から管理者がEmail連携設定をした状態でQueryPie内部DBにユーザーアカウント認証情報がある場合、ユーザーはEmailを通じて自分のパスワードをリセットできます。ID ProviderやLDAPのように認証情報がQueryPie外部にある場合、この機能は使用できません。

<Callout type="info">
**QueryPieでユーザーのパスワードをリセットする方法**
* 管理者がユーザーのパスワードを直接リセット（一時パスワード発行）
* 管理者がユーザーにパスワードリセットメールを送信（QueryPie 10.2.2からサポート）
* ユーザーがログイン画面で直接パスワードリセットを要求（QueryPie 10.2.2からサポート）
</Callout>

### パスワードリセットメールを通じたパスワードリセット

管理者からユーザーのパスワードリセットについて通知を受けた場合、自分のメールの受信トレイにパスワードリセットメールが受信されているか確認する必要があります。ユーザーが受信するメールの内容は以下のようなものです。

メールで提供されるパスワードリセットリンクは30分間有効です。この場合、管理者にパスワードリセット要求を再度する必要があります。

### ログイン画面で直接パスワードリセット用認証コード要求

1. ログインページの「Forgot your password?」リンクをクリックします。<br/>
2. ダイアログで自分のIDを入力し`Request`ボタンをクリックします。<br/>
3. 自分のメールの受信トレイで本人確認のための認証コードが受信されているか確認します。<br/> 
4. メールで確認した認証コードを入力します。受信した認証コードは30分以内に入力する必要があります。メールを受信できなかった場合は「Resend authentication code」リンクを押して認証コードメールを再要求できます。ただし、再送信要求は最初の再送信後5分経過してから可能です。<br/> 
5. 自分のIDを入力し新しいパスワードを登録します。<br/>  
6. 変更されたパスワードでログインします。
