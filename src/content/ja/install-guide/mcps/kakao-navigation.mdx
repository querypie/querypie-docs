# Kakao Navigation

Kakaoナビゲーション APIを通じて、経路案内、道案内、経由地設定などのナビゲーション機能を使用できます。

## 主な機能
- 出発地から目的地までの経路案内
- 経由地を含む経路計画
- 複数出発地経路計算
- 複数目的地経路計算
- 未来時間基準経路計算

## 基本情報

### 認証方式
API Keyを使用して認証します。

### 入力パラメータ
- **API Key**（必須）：Kakao開発者センターで発行されたREST API キー

## サポートするMCPツール

### 経路案内ツール
#### navigation_directions
出発地から目的地までの経路を計算します。

**入力パラメータ：**
- `origin`（必須）：出発地座標（経度,緯度）
- `destination`（必須）：目的地座標（経度,緯度）
- `priority`（オプション）：経路優先順位（RECOMMEND、TIME、DISTANCE）
- `car_fuel`（オプション）：車両燃料タイプ
- `car_hipass`（オプション）：ハイパス使用の有無
- `alternatives`（オプション）：代替経路提供の有無
- `road_details`（オプション）：道路詳細情報提供の有無

#### navigation_waypoints
経由地を含む経路を計算します。

**入力パラメータ：**
- `origin`（必須）：出発地座標
- `destination`（必須）：目的地座標
- `waypoints`（必須）：経由地座標リスト
- `priority`（オプション）：経路優先順位
- `car_fuel`（オプション）：車両燃料タイプ
- `car_hipass`（オプション）：ハイパス使用の有無

#### navigation_origins
複数の出発地から一つの目的地までの経路を計算します。

**入力パラメータ：**
- `origins`（必須）：出発地座標リスト
- `destination`（必須）：目的地座標
- `priority`（オプション）：経路優先順位
- `car_fuel`（オプション）：車両燃料タイプ
- `car_hipass`（オプション）：ハイパス使用の有無

#### navigation_destinations
一つの出発地から複数の目的地までの経路を計算します。

**入力パラメータ：**
- `origin`（必須）：出発地座標
- `destinations`（必須）：目的地座標リスト
- `priority`（オプション）：経路優先順位
- `car_fuel`（オプション）：車両燃料タイプ
- `car_hipass`（オプション）：ハイパス使用の有無

#### navigation_future
未来時間を基準に経路を計算します。

**入力パラメータ：**
- `origin`（必須）：出発地座標
- `destination`（必須）：目的地座標
- `departure_time`（必須）：出発時間
- `priority`（オプション）：経路優先順位
- `car_fuel`（オプション）：車両燃料タイプ
- `car_hipass`（オプション）：ハイパス使用の有無

## 使用例
Kakaoナビゲーション MCPサーバーは、Kakao地図 MCPサーバーと一緒に活用する必要があります。<br/>
出発地から目的地までの経路を見つけるために、まず場所クエリを座標に変換し、これを利用して経路を計算します。

例えば、
- プロンプトで「江南駅から弘大入口駅まで行く道を教えて」と入力すると、
- LLMはまずKakao地図 MCPサーバーの`map_address_to_coord`ツールを呼び出し、江南駅の座標、弘大入口駅の座標を取得します。
- 続いてLLMはKakaoナビゲーション MCPサーバーの`navigation_directions`ツールを呼び出し、経路を計算します。
- そして、ツール呼び出し結果に基づいてチャットで応答します。

## API Key設定方法
1. Kakao開発者センターでアプリケーションを登録します。
2. REST API キーを発行します。
3. QueryPie AIでKakao Navigation統合を選択します。
4. 発行されたAPI キーを入力して接続を完了します。

## 注意事項
- 生成したAPI Keyは安全に保管してください。
- 1日の呼び出し制限を確認し、適切に使用してください。
- 座標はWGS84座標系を使用します。
- リアルタイム交通情報を反映した経路を提供します。
