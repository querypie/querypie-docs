# Atlassian Confluence

Atlassian Confluenceと接続してチーム協業と文書化プラットフォームを管理できます。

## 主な機能
- ページとブログ投稿の管理
- スペースと権限の管理
- コンテンツの検索と照会
- コメントと協業機能
- 添付ファイルの管理

## 基本情報

### 認証方式
APIトークンを使用して認証します。

### 入力パラメータ
- **API Token** (必須): Confluenceで生成されたAPIトークン
- **Base URL** (必須): ConfluenceインスタンスURL
- **Username** (必須): Confluenceユーザー名

## サポートされているMCPツール

### get_spaces
すべてのスペースのリストを返します。結果はIDの昇順でソートされ、`limit`パラメータで結果数を制限できます。

### get_pages
すべてのページのリストを返します。結果数は`limit`パラメータで制限でき、追加の結果は`Link`レスポンスヘッダーの`next` URLで確認できます。

### get_page_by_id
特定のページの詳細情報を照会します。

### bulk_user_lookup
リクエストボディで提供されたIDに対するユーザー詳細情報を返します。

### create_page
スペースに新しいページを作成します。

### update_page
IDでページを更新します。

### delete_page
IDでページを削除します。

## 使用例

例えば、
- プロンプトで「特定のスペースのページリストを照会して」と入力すると、
- LLMはMCPを通じて`get_pages`ツールを呼び出してページリストを照会します。
- そして、ツール呼び出し結果に基づいてチャットで応答します。

## APIトークン生成方法
1. Confluence設定 → Personal Access Tokensに移動
2. "Create token"をクリックして新しいトークンを生成
3. 必要な権限を設定します。

## 注意事項
- 生成したAPIトークンは安全に保管してください。
- 必要最小限の権限のみを付与することを推奨します。
- Base URLはHTTPSを使用することを推奨します。
