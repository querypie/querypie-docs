# インストール

QueryPie ACP製品をインストールする方法はContainer実行方式、インストール支援プログラムのバージョンによって案内文書が区分されて提供されます。

### Compose ToolでContainerを実行する

一般的なインストール/実行方法であり、製造元で推奨する方式です。
Docker Composeを活用してContainer Imageをダウンロードし、実行/終了します。

この場合、インストール支援プログラムのバージョンによって、2つのインストール方式が提供されます。

#### setup.v2.shでインストールする

2025年8月リリースされたQueryPie ACP Community Editionとともに最終顧客社とパートナーエンジニアに提供されるインストール支援プログラムです。
Docker Hubに公開されたContainer Imageをダウンロードしてインストールする方式です。

インストール過程が自動化されており、初めてインストールする利用者も製品を簡単にインストールできます。
setup.v2.shが実際に実行するインストール過程はsetup.shのインストール方式と同等です。
setup.v2.shはその過程が自動化されておりVersion 10などの旧バージョンインストールをサポートしないという違いがあります。
より詳細な違いはこの文書を参照してください：[setup.shとsetup.v2.shの比較](installation/comparison-of-setupsh-and-setupv2sh)

Apple Siliconを使用するmacOS環境のインストールをサポートします。

詳細なインストール方法はこの文書を参照してください：[インストールガイド - setup.v2.sh](installation/installation-guide-setupv2sh)

#### setup.shでインストールする

QueryPie ACP Version 9時代からパートナーエンジニアに提供されたインストール支援プログラムです。
HarborというContainer Registryでアカウントを発行してContainer Imageをダウンロードできます。

詳細なインストール方法はこの文書を参照してください：[インストールガイド - 簡単な構成](installation/installation-guide-simple-configuration)

### Kubernetes環境にインストールする

QueryPie ACPはContainer環境で動作するソフトウェア製品です。
Compose Toolを利用してインストール/実行するのが一般的ですがKubernetes環境でもスムーズに動作します。

Kubernetes環境で製品をインストールする場合、AWS EKS、GCP GKEなどのManaged Kubernetes ClusterとSelf-Hosted Clusterをすべて使用できます。

* AWS EKS環境：[AWS EKS環境でインストールする](installation/installing-on-aws-eks)
* Helm Chart：[https://github.com/chequer-io/querypie-deployment](https://github.com/chequer-io/querypie-deployment)
