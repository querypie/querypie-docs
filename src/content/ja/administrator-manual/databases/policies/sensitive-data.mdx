## Overview

組織内で個人情報または機密情報のように管理が必要なデータを機密データポリシーとして設定する方法を確認できます。Sensitive Dataメニューでポリシーを設定するコネクションを選択してポリシーを生成し、機密なデータをテーブル、カラム基準で規則を登録し管理できます。ポリシーとして生成された場合、別途ログを通じて機密データにアクセスした記録を確認でき、Alert機能を通じて通知を受けることもできます。


## 機密データポリシー生成

新しい機密データポリシーをコネクション基準で登録します。

<figure data-layout="center" data-align="center">
![Administrator &gt; Databases &gt; Policies &gt; Sensitive Data](/administrator-manual/databases/policies/sensitive-data/image-20240729-054311.png)
<figcaption>
Administrator &gt; Databases &gt; Policies &gt; Sensitive Data
</figcaption>
</figure>

1. Database設定メニューでPolicies > Sensitive Dataメニューに移動します。
2. 右上の`Create Policy`ボタンをクリックします。
3. ポリシー生成のための以下の情報を入力します。
    1.  **Policy Name**  ：ポリシーを画面で区別できる名前です。
    2.  **Target Connection**  ：ポリシーを適用するコネクションを選択します。1個のポリシー当たり1個のコネクションを接続できます。
4. `Save`ボタンで保存します。

該当ポリシーがSensitive Dataポリシーリストに生成されたことを確認できます。


## 機密データポリシーに規則登録

ポリシーを生成した後、実際にポリシーを適用するデータの経路を規則として登録します。

<figure data-layout="center" data-align="center">
![Administrator &gt; Databases &gt; Policies &gt; Sensitive Data > Rule List](/administrator-manual/databases/policies/sensitive-data/image-20240820-120656.png)
<figcaption>
Administrator &gt; Databases &gt; Policies &gt; Sensitive Data > Rule List
</figcaption>
</figure>

1. Sensitive Dataメニューで生成したポリシーをクリックします。
2. ポリシー詳細情報および規則登録画面が表示され、右側で`Add Rule List`ボタンをクリックします。
3. ポリシーを適用するデータの経路を順番に選択します。
    1.  **Database Name**  ：規則登録のための必須値です。
    2.  **Table Name**  ：規則登録のための必須値です。この場合、該当テーブルデータ全体が機密データとして設定されます。
    3.  **Column Name**  ：選択したテーブル内の特定カラムにのみポリシーを適用したい場合カラムを選択します。
4. ポリシーを適用するデータの機密度レベルを選択します。
    1. High、Medium、Low 3つのレベル中1つを選択します。
    2. 選択したSensitive Level情報は今後Alert発送のトリガー条件として設定できます。
5. `Ok`ボタンで保存します。

Rule Listタブで規則が登録されたことを確認できます。これでユーザーが該当データにアクセスする場合、Logsタブでアクセス履歴を確認できます。機密データポリシーを通知として設定しようとする場合、[Alerts](../../general/company-management/alerts)文書を参照してください。
