## Overview

<Callout type="info">
Multi-Factor Authentication (MFA)とは？<br/>パスワード以外に追加で認証情報を入力する多重認証セキュリティ技法です。<br/>追加認証手段として物理/仮想OTP（One Time Password：1回用パスワード生成器）、生体認証、Emailなどを使用することもあります。
</Callout>

QueryPieは管理者ページで2次認証手段としてGoogle OTPまたはEmailを指定できます。

## MFA手段指定

<Callout type="important">
MFA手段を指定するにはAuthentication TypeがInternal databaseまたはLDAPで選択されていなければなりません。<br/>EmailをMFA手段として指定するにはまずEmail Integration設定が完了されていなければなりません。
</Callout>

* Admin &gt; General &gt; User Management &gt; Authenticationに移動します。
* Authentication TypeがInternal databaseの場合<br/>
* Authentication TypeがLDAPの場合<br/>

## サーバー接続時MFA使用

<Callout type="info">
この設定はサーバーアクセス制御（SAC）ライセンスが有効化されていなければ使用できません。
</Callout>

* Admin &gt; General &gt; Company Management &gt; Securityに移動します。
* Server Connection Security項目下のAccess Server with MFAを"Enable"に設定しMFA TypeをGoogle OTPまたはEmailを選択します。（Access Server with MFAはデフォルト値がDisableです。<br/>
