# Policy Exception logs

### Overview

政策例外処理の状態変化を記録する機能です。政策例外が作成、修正、削除、期限切れ、有効化、無効化されるたびに該当イベントをログとして記録して追跡できます。リストで行をクリックすると特定の政策例外の詳細内容を見ることができます。

### Policy Exception Logsに記録される内容

*  **Action At** : Eventが発生した時点
*  **Event Type**  : イベントのタイプ別区分です。
    *  **Created**  : Workflowまたは管理者によって政策例外が作成されたイベント
    *  **Modified** : 管理者によって政策例外が修正されたイベント
    *  **Deleted**  : 管理者が特定の政策例外を手動で削除したイベント
    *  **Expired**  : Workflowまたは管理者が指定した期限が到来して政策例外が期限切れになったイベント
    *  **Activated**  : 管理者によって特定の政策例外が有効化状態になったイベント
    *  **Deactivated**  : 管理者によって特定の政策例外が無効化（Inactive）状態になったイベント
*  **Exception Type**  : 現在存在する政策例外はRestricted Data Access、Unmasking二つのタイプがあります。
*  **Allowed Users**  : 政策例外対象ユーザーまたはグループが記録されます。また、管理者が指定したユーザーの属性（Attribute of Users）が記録されます。
*  **Description** : 政策例外に設定された説明です。
*  **Excepted By**  : 政策例外を発生させた主体です。
    *  **Workflow**  : 政策例外要求および承認がworkflowを通じて行われた場合workflowで記録されます。
    *  **User(Admin)**  : 政策例外が管理者によって手動で作成された場合ユーザーDisplay Nameが表示されます。
*  **Start Time** : 政策例外の開始時点です。
*  **End Time**  : 政策例外の終了時点です。
*  **Action By :** 既存政策例外の修正、削除、ActiveまたはInactiveに状態変更、期限切れを発生させた主体です。（期限切れ主体は期限切れ時点に発生するため常にSystemで記録されます。）
*  **Exception Name :** workflowで実行された場合workflowタイトルが表示されます。管理者の手動政策例外の場合Exception Nameが表示されます。
